Lesson m5-1 ended with a 7.8 dB indictment: uncoded QPSK needs 9.6 dB Eb/N₀ for a working link, while Shannon says 2 bit/s/Hz is achievable at 1.8 dB. This lesson is about the machinery that closes that gap — forward error correction — and about the operational structure built on top of it: the MODCOD table, a ladder of modulation-plus-code-rate combinations, each with a precise signal-quality threshold, that lets a link trade throughput for robustness in fine steps and in real time.

The MODCOD table is the single most consequential object in this module. It is the lookup that converts the Module 4 link budget's output (achieved C/N₀) into an information rate. The spine project gains that lookup in this lesson: Stage 2 attaches a DVB-S2 MODCOD table and selection logic to the Stage-1 carrier model, and demonstrates the defining behaviour of a modern satellite link — throughput that degrades gracefully through a rain fade instead of failing.

One warning applies throughout. MODCOD thresholds are quoted in the literature in three different currencies — Eb/N₀, Es/N₀, and required C/N₀ — and figures in the last currency are meaningless unless the symbol rate is stated alongside them, because required C/N₀ = Es/N₀ + 10·log₁₀(R_s). A great deal of confusion in datasheets and marketing material traces to comparing dBHz figures computed at different, unstated symbol rates. This lesson fixes the conversions once, precisely.

1. The forward error correction principle

Forward error correction (FEC) adds structured redundancy to the transmitted bit stream. An encoder takes k information bits and produces n > k coded bits; the ratio r = k/n is the code rate. The receiver's decoder exploits the redundancy to identify and correct errors introduced by noise — without retransmission, hence "forward."

The cost is rate: at code rate 1/2, half of every transmitted bit is parity, so the information rate is half the raw bit rate of m5-1:

R_info = r · R_b = r · log₂(M) · R_s

The benefit is coding gain: the reduction in required Eb/N₀ to achieve a target error rate, compared to the uncoded system. Modern codes buy 5–9 dB of coding gain for their rate cost — and section 6 shows this is a spectacular trade, because the gain lands within 1–2 dB of the Shannon bound.

The intuition for why redundancy beats raw repetition: a good code spreads each information bit's influence across many transmitted bits in a pseudo-random pattern, so that noise must corrupt a coordinated set of symbols — not just one — to destroy information. The decoder then performs probabilistic inference over the whole block. Long blocks average over the noise; DVB-S2's normal frame is 64,800 coded bits precisely because length is where the performance comes from.

2. Code families, and why LDPC won

Three generations of codes carried satellite communications to the Shannon bound:

Convolutional codes with Viterbi decoding (1960s–1990s): the classical workhorse, typically concatenated with an outer Reed–Solomon code. Coding gain around 5–6 dB; a gap to Shannon of 3–5 dB remained. This is the DVB-S generation.

Turbo codes (1993): two convolutional encoders and an interleaver, decoded iteratively. The first practical near-capacity codes — within ~1 dB of Shannon — and the shock that reset the field. Adopted in 3G/4G cellular and several satellite systems.

Low-density parity-check (LDPC) codes: invented by Gallager in 1962, forgotten for thirty years because the decoding was computationally infeasible, then rediscovered in the 1990s. LDPC codes match turbo performance with a decisive practical advantage: the belief-propagation decoder is massively parallelisable, which suits hardware implementation at high symbol rates. DVB-S2 (2005) selected LDPC with a small outer BCH code to clean up the error floor, and essentially every subsequent satellite standard — DVB-S2X, 5G NR (and therefore 3GPP NTN, m5-5) — followed.

The DVB-S2 FEC structure, which the rest of this module assumes: information bits → BCH outer encoder → LDPC inner encoder → a FECFRAME of 64,800 bits (normal) or 16,200 bits (short) → bit interleaver → constellation mapper. Eleven code rates from 1/4 to 9/10 are defined; combined with the four DVB-S2 constellations they produce the MODCOD table.

3. The threshold: quasi-error-free

Every MODCOD in the table carries one number: the minimum signal quality at which its decoder achieves quasi-error-free (QEF) operation. DVB-S2 defines QEF as a frame error rate better than 10⁻⁷ — one lost frame per ten million, roughly one error event per hour at typical rates. The practical meaning: below threshold the LDPC decoder fails frequently and the link is unusable; above it, errors are so rare the link behaves as a clean pipe.

This cliff-edge behaviour is characteristic of capacity-approaching codes. Uncoded links degrade gradually — BER worsens smoothly as Eb/N₀ falls. LDPC-coded links have a waterfall: over a fraction of a dB, the frame error rate moves from negligible to total. The threshold is therefore sharp and meaningful in a way an uncoded "required Eb/N₀" never was, and it is why link adaptation (section 7) can be built on simple threshold comparisons.

Thresholds are specified as Es/N₀ — energy per symbol over noise density, equivalently C/N measured in the symbol-rate bandwidth. Es/N₀ relates to the other currencies as:

Es/N₀ [dB] = Eb/N₀ [dB] + 10·log₁₀(η) where η = r · log₂(M) bits per symbol
C/N₀ [dBHz] = Es/N₀ [dB] + 10·log₁₀(R_s)

4. The MODCOD table

An excerpt of the DVB-S2 MODCOD table (ETSI EN 302 307-1, Table 13: normal FECFRAME, AWGN channel, ideal demodulation). η is the spectral efficiency in bits per symbol, including the FEC but not framing overhead (m5-4's business):

QPSK 9/101.7896.42

16APSK 5/63.30011.61

Read the table as a ladder. Each rung upward buys spectral efficiency and costs threshold — and the two currencies move together in remarkably uniform steps: adjacent rungs are typically 0.5–1.5 dB and 0.1–0.3 bit/symbol apart. This fine granularity is deliberate. It means that whatever C/N₀ a link achieves, there is a rung close beneath it, and little signal quality is wasted as unusable margin. DVB-S2X extended the ladder further — more rungs (finer steps), 64/128/256-APSK at the top, and very-low-SNR MODCODs down to −10 dB Es/N₀ at the bottom for mobile and D2D applications (m5-6).

Note two structural features. QPSK 9/10 (η = 1.789) and 8PSK 2/3 (η = 1.981) sit almost adjacent in threshold (6.42 vs 6.62 dB): near the crossover points, a denser constellation with a stronger code and a sparser constellation with a weaker code are nearly interchangeable, and the choice is made by secondary effects — amplifier back-off (m5-1 section 5) favouring the sparser constellation. And the ladder's top rung is not "best": 32APSK 9/10 exists for gateway feeder links with large antennas and linearised amplifiers, not for consumer terminals that will rarely see 16 dB Es/N₀.

5. From threshold to required C/N₀ — always with a symbol rate

Converting the table to the link budget currency requires the symbol rate. For the 208.3 Msym/s carrier established in m5-1 (250 MHz, α = 0.20), 10·log₁₀(R_s) = 83.2 dBHz:

MODCOD Es/N₀ (dB) Required C/N₀ (dBHz) Information rate
QPSK 1/21.0084.2206.0 Mbps
8PSK 3/47.9191.1464.1 Mbps
16APSK 3/410.2193.4618.0 Mbps
32APSK 3/412.7395.9771.4 Mbps

Two observations. First, the entire usable ladder spans about 12 dB of C/N₀ (84 to 96 dBHz at this symbol rate) — this is the operating window the Module 4 link budget must be designed against, and it is why the m4-7 sensitivity analysis mattered: a dB gained anywhere in the budget converts directly into roughly one rung of the ladder, which is 5–15% more throughput.

Second — the warning from the introduction, now concrete: these dBHz figures are properties of this carrier's symbol rate. Halve the symbol rate and every required C/N₀ drops by 3 dB while every information rate halves. A required-C/N₀ figure without a stated symbol rate is not a specification; it is noise. When this module's numbers are cross-referenced against Module 4's worked examples, the reconciliation must always pass through R_s.

6. The gap to Shannon, closed

m5-1 measured uncoded QPSK's gap to the Shannon bound at 7.8 dB. Repeating the measurement for the coded table — comparing each MODCOD's Es/N₀ threshold against the Shannon minimum SNR for its spectral efficiency, 10·log₁₀(2^η − 1):

MODCOD η Threshold (dB) Shannon minimum (dB) Gap
QPSK 1/20.9891.00−0.071.07 dB
QPSK 3/41.4874.032.561.47 dB
8PSK 2/31.9816.624.691.93 dB
16APSK 3/42.96710.218.341.87 dB
32APSK 3/43.70312.7310.801.93 dB

The 7.8 dB uncoded gap has collapsed to 1–2 dB across the whole ladder. This is the quantitative meaning of "modern coding is near-capacity," and it carries a strategic implication worth stating bluntly: the physical layer is essentially finished. There is no future code that will find another 5 dB, because Shannon says the 5 dB does not exist. System capacity gains now come from elsewhere — more spectrum (m4-3), more aggressive frequency reuse and beam density (Module 3's payload), tighter roll-off (m5-1), and less overhead (m5-3, m5-4). An architect evaluating a vendor's claimed "revolutionary waveform" should reach for this table first.

7. ACM: the ladder in motion

The MODCOD table is static; the channel is not. Module 4 established that a Ka-band link's C/N₀ moves over tens of dB-seconds — rain fades of 5–15 dB developing over seconds to minutes (m4-4), slant-range variation of several dB across a LEO pass (m4-1, m4-7). Three operating disciplines exist for coping:

CCM — constant coding and modulation: one MODCOD, chosen so the link closes at the worst tolerated condition. Simple, and wasteful: in clear sky — the overwhelming majority of the time — the link runs far above threshold and the excess C/N₀ buys nothing.

VCM — variable coding and modulation: the MODCOD varies on a fixed schedule (per service, per time slot), not in response to the channel. Used where no return channel exists to report conditions — broadcast, some downlink-only Earth observation.

ACM — adaptive coding and modulation: the receiver continuously estimates its Es/N₀, reports it over the return channel, and the transmitter selects the highest rung whose threshold the reported quality clears (plus margin). This is the operating mode of every modern broadband satellite system, and it converts the CCM's wasted clear-sky margin directly into throughput.

The control loop's engineering details decide whether ACM works well:

  • Measurement and reporting delay: the loop must outrun the fade. Rain fade slopes at Ka-band are commonly 0.1–1 dB/s; LEO geometry adds slower, predictable variation. Reporting intervals are tens of milliseconds to seconds. If the fade outruns the loop, frames are lost at the old MODCOD before the step-down lands — this is the ACM response-latency problem posed in m4-7's reflection.
  • Hysteresis: step down at threshold + margin_down, step back up only at threshold + margin_up, with margin_up > margin_down. Without hysteresis, a link sitting at a rung boundary oscillates between MODCODs every reporting interval, and every switch has a cost.
  • Margin policy: the selection margin (typically 0.5–2 dB above the ideal threshold) absorbs estimation error, loop delay, and implementation loss (section 8). Larger margin means fewer decoder failures and less throughput. This single parameter is a direct availability-versus-capacity dial.

The system-level consequence is the defining behaviour of modern satellite broadband: graceful degradation. Through a 9 dB rain fade, a 618 Mbps 16APSK-3/4 carrier steps down the ladder and emerges at QPSK 1/2 carrying 206 Mbps — one-third the throughput, zero loss of service. The uncoded, un-adaptive link of m5-1 would simply have died. The spine project's Stage-2 demonstration is exactly this sweep.

8. Implementation margin

The Table 13 thresholds are ideal: AWGN channel, perfect synchronisation, infinite-precision decoding. Real modems miss them by an implementation margin — typically 0.5–1.5 dB, covering imperfect carrier and timing recovery, quantisation, phase noise, and residual amplifier distortion after the back-off and pre-distortion of m5-1. System engineering practice adds this margin to every threshold before it enters a link budget or an ACM selection rule.

This is also where the m5-1 amplifier discussion closes its loop: the nonlinear distortion that APSK tolerates and QAM does not shows up here, as a modulation-dependent implementation margin. The DVB-S2 implementation guidelines (ETSI TR 102 376) quantify the total degradation over a nonlinear channel per MODCOD — the higher rungs lose more — and a careful system model uses those degraded thresholds, not the ideal ones.

10. What this lesson delivers to the design chain

The link budget's C/N₀ is now convertible into bits. Given an achieved C/(N₀+I₀) from the Module 4 engine and a symbol rate from the m5-1 carrier model, the MODCOD table selects the operating point and yields an information rate — and under ACM, that conversion re-runs continuously as the channel moves, producing throughput that bends through fades instead of breaking.

The spine project's Stage 2 implements exactly this: a ModCodTable with the DVB-S2 thresholds, a selection rule with configurable margin, and a fade-sweep demonstration of graceful degradation. What the engine still ignores: the carrier is assumed to serve one user with zero framing cost. m5-3 introduces the sharing problem — multiple access and scheduling, where a beam's information rate is divided among hundreds of terminals — and m5-4 charges the framing, pilot, and signalling overhead that separates the information rate computed here from the throughput a user actually measures.

    ("QPSK 3/4",   1.487473,  4.03),
    ("8PSK 3/4",   2.228124,  7.91),
    ("16APSK 3/4", 2.966728, 10.21),
    ("32APSK 3/4", 3.703295, 12.73),
]

def ebn0(esn0_db, eta):    return esn0_db - 10*math.log10(eta)

        print(f"{name:<12}{es:>7.2f} {ebn0(es,eta):>7.2f} "
              f"{req_cn0(es,rs):>9.1f} {eta*rs/1e6:>9.1f} Mbps")

    ("QPSK 1/4",   0.490243, -2.35), ("QPSK 1/2",   0.988858,  1.00),

]

def select(esn0_db, margin_db):
    """Highest-eta MODCOD whose threshold + margin is met."""
    best = None
    for name, eta, thr in MODCODS:
        if esn0_db >= thr + margin_db and (best is None or eta > best[1]):
            best = (name, eta, thr)
    return best

RS = 208.3e6

              f"{eta*RS/1e6:>7.1f} Mbps  (headroom {achieved-thr:.2f} dB)")
    else:
        print(f"  margin {margin:>3.1f} dB -> no MODCOD selectable")

Exercise 3 — Spine Stage 2: MODCOD selection and the fade sweep

The throughput engine gains its lookup: achieved C/N₀ in, information rate out — and the graceful-degradation demonstration.

import math

# ═══════════════════════════════════════════════════════
# M5 SPINE PROJECT — THROUGHPUT ENGINE
# Stage 2: MODCOD table + ACM selection.
#   Stage 1 (m5-1) gave: bandwidth, roll-off -> symbol rate.
#   Stage 2 adds: achieved C/N0 -> MODCOD -> information rate.
# Next stages: m5-3 access overhead, m5-4 framing overhead,
#              m5-5 NTN branch, m5-6 D2D, m5-7 full assembly.
# ═══════════════════════════════════════════════════════

MODCOD_TABLE = [
    # (name, eta bit/sym, EsN0 threshold dB) — EN 302 307-1 Table 13
    ("QPSK 1/4",    0.490243, -2.35),
    ("QPSK 1/2",    0.988858,  1.00),
    ("QPSK 3/4",    1.487473,  4.03),
    ("QPSK 9/10",   1.788612,  6.42),
    ("8PSK 3/4",    2.228124,  7.91),
    ("16APSK 2/3",  2.637201,  8.97),
    ("16APSK 3/4",  2.966728, 10.21),
    ("16APSK 5/6",  3.300184, 11.61),
    ("32APSK 3/4",  3.703295, 12.73),
    ("32APSK 9/10", 4.453027, 16.05),
]

class ThroughputEngine:
    """
    Stage-2 engine: carrier geometry (Stage 1) + MODCOD selection.
    Information rate only — access and framing overhead arrive in
    m5-3/m5-4 and will strictly reduce these numbers.
    """
    def __init__(self, name, bw_hz, alpha, impl_margin_db=1.0,
                 select_margin_db=0.5):
        self.name  = name
        self.bw_hz = bw_hz
        self.alpha = alpha
        self.impl_margin   = impl_margin_db     # modem reality vs Table 13
        self.select_margin = select_margin_db   # ACM safety margin

    @property
    def symbol_rate(self):
        return self.bw_hz / (1 + self.alpha)    # Stage 1

    def esn0_from_cn0(self, cn0_dbhz):
        return cn0_dbhz - 10 * math.log10(self.symbol_rate)

    def select_modcod(self, cn0_dbhz):
        """Highest-eta MODCOD supportable at this C/N0, or None."""
        esn0 = self.esn0_from_cn0(cn0_dbhz)
        need = self.impl_margin + self.select_margin
        best = None
        for name, eta, thr in MODCOD_TABLE:
            if esn0 >= thr + need and (best is None or eta > best[1]):
                best = (name, eta, thr)
        return best

    def info_rate(self, cn0_dbhz):
        sel = self.select_modcod(cn0_dbhz)
        return sel[1] * self.symbol_rate if sel else 0.0

    def report(self, cn0_dbhz):
        esn0 = self.esn0_from_cn0(cn0_dbhz)
        sel  = self.select_modcod(cn0_dbhz)
        print(f"\n{'─'*58}")
        print(f"  THROUGHPUT ENGINE — {self.name}")
        print(f"{'─'*58}")
        print(f"  Symbol rate        {self.symbol_rate/1e6:>10.1f} Msym/s")
        print(f"  Achieved C/N0      {cn0_dbhz:>10.1f} dBHz")
        print(f"  Achieved Es/N0     {esn0:>10.2f} dB")
        print(f"  Margins (impl+sel) {self.impl_margin:>6.1f} + {self.select_margin:.1f} dB")
        if sel:
            name, eta, thr = sel
            print(f"  Selected MODCOD    {name:>10}  (thr {thr:.2f} dB)")
            print(f"  INFORMATION RATE   {eta*self.symbol_rate/1e6:>10.1f} Mbps")
        else:
            print(f"  Selected MODCOD          NONE — link does not close")
        print(f"{'─'*58}")

# ── The graceful-degradation sweep ──────────────────────
# Clear-sky C/N0 from the Module 4 engine, then a rain fade
# develops. Watch the ladder do its job.
eng = ThroughputEngine("Ka forward, user beam", bw_hz=250e6, alpha=0.10)

clear_sky_cn0 = 96.0
print(f"Rain fade sweep — clear-sky C/N0 = {clear_sky_cn0} dBHz")
print(f"{'fade (dB)':>10}{'C/N0':>8}{'Es/N0':>8}{'MODCOD':>14}{'Mbps':>9}")
for fade in [0, 2, 4, 6, 8, 10, 12, 14, 16]:
    cn0  = clear_sky_cn0 - fade
    esn0 = eng.esn0_from_cn0(cn0)
    sel  = eng.select_modcod(cn0)
    name = sel[0] if sel else "— LINK DOWN"
    rate = eng.info_rate(cn0) / 1e6
    print(f"{fade:>10}{cn0:>8.1f}{esn0:>8.2f}{name:>14}{rate:>8.1f}M")

eng.report(cn0_dbhz=96.0)

The fade sweep shows throughput stepping down smoothly through a fade — from the network's perspective. From a single user's perspective, their allocated capacity during the fade depends on how the scheduler (m5-3) redistributes the shrunken beam capacity. Decide: during a beam-wide fade, should the scheduler preserve per-user fairness (everyone degrades equally) or protect a premium tier (some users see no change, others lose service)? Defend your choice, and identify what contractual instrument encodes it.

PROMPT 04

VCM exists for links with no return channel. A LEO Earth-observation constellation downlinks imagery over high-latitude ground stations with a strong, predictable elevation profile per pass. Design a VCM schedule for one pass in outline: what drives the MODCOD change points, how would you compute them from the m2-3/m2-4 pass geometry plus the m4 link budget, and what residual inefficiency does VCM carry versus true ACM on this specific link?
