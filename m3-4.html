Multibeam Antennas & Phased Arrays

MODULE 3

The antenna is the interface between the satellite and the world it serves. Everything computed in Module 2 — coverage geometry, elevation angles, Doppler shifts — arrives at the antenna as a demand on its performance: generate a beam of a specific width, pointed at a specific ground cell, with a specific gain, while simultaneously generating dozens or hundreds of other beams. This lesson covers the physics of that progression — from single-beam reflector to multibeam array to electronically steered phased array — and the array theory (gain, beam steering, grating lobes, sidelobe control) that underlies the link budget in Module 4.

1. The Single-Beam Reflector: The Baseline

The parabolic reflector has been the workhorse of satellite communications since the beginning. A feed at the focal point illuminates a dish; the dish collimates the energy into a narrow beam. The two governing relationships are:

G = η · (π D / λ)²  ·  HPBW ≈ 70λ / D (degrees)

where η is aperture efficiency (0.55–0.70), D is dish diameter, and λ is wavelength. Larger aperture gives higher gain and narrower beam. At Ka-band (28 GHz, λ = 10.7 mm), a 1-metre dish achieves approximately 42 dBi gain and 0.75° beamwidth.

On a GEO satellite a reflector points at a fixed region and stays there. On a LEO satellite moving at 7.6 km/s, a mechanically steered reflector must rotate continuously — feasible for single-beam applications but impractical for a multibeam payload generating dozens of simultaneous spot beams.

2. From Single-Beam to Multibeam: The Reflector Array Feed

The first step toward multibeam operation is the multibeam reflector: a single large reflector dish illuminated by a focal-plane array of feed elements. Each feed element generates one beam; displaced from the reflector axis, its beam points at a different ground location. N feed elements generate N simultaneous spot beams from a single reflector aperture.

This approach — used on first-generation HTS GEO satellites (ViaSat-1, Jupiter, SES O3b) — shares a single mechanical structure across all beams and requires no moving parts for normal operation. The limitation: beams cannot be steered. Each feed position maps to a fixed ground position. For LEO, the satellite's motion means the ground coverage region moves continuously — requiring either rotation of the whole dish or a different architecture.

3. The Phased Array: Principle of Operation

The phased array replaces the reflector with an array of radiating elements, each connected to a module that independently controls the amplitude and phase of its signal. By adjusting phases across the array, element contributions add constructively in the beam direction and destructively elsewhere.

For a linear array of N elements spaced d apart, the phase shift applied to element n to steer the beam to angle θ₀ from broadside is:

φₙ = −n · k · d · sin θ₀    where k = 2π/λ

This compensates for the path-length difference n·d·sin θ₀ between elements, making all contributions arrive in phase at the desired direction.

Array gain

The maximum gain of a uniformly illuminated planar array of N elements is G_array = N · G_element. For a half-wavelength spaced patch array (G_element ≈ 6 dBi), a 256-element array achieves 6 + 24 = 30 dBi; a 2,048-element array achieves 6 + 33 = 39 dBi. Real arrays achieve approximately 60–70% of the theoretical maximum due to element pattern roll-off, mutual coupling, and quantisation errors.

Grating lobes and sidelobe control

Element spacing d must satisfy d ≤ λ/2 to avoid grating lobes — secondary maxima at angles satisfying d·sin θ = nλ for n ≠ 0, appearing at the same gain as the main beam. Wider spacing reduces element count for a given aperture but limits the usable scan range.

Sidelobe level depends on the amplitude weighting: uniform illumination gives sidelobes at approximately −13 dB relative to peak. Taylor or Chebyshev amplitude tapering reduces sidelobes to −25 to −40 dB at the cost of slightly wider beamwidth and ~1–2 dB gain reduction.

4. Active Phased Arrays: The T/R Module

In an active phased array (AESA), each element (or small group) has its own transmit/receive (T/R) module containing an LNA, power amplifier (PA), phase shifter, and amplitude control. Distributed amplification eliminates the central TWTA, provides graceful degradation (losing 5% of elements reduces gain by less than 0.5 dB for a 1,000-element array), and enables precise per-element amplitude and phase control for sidelobe shaping and null steering.

The disadvantage: each T/R module adds cost, mass, and power. A 1,000-element Ka-band AESA may have T/R modules consuming 2–5 W each — 2–5 kW total for transmit. GaN (gallium nitride) power amplifiers at Ka-band achieve 30–40% power-added efficiency (PAE), meaning 60–70% of the DC input power is dissipated as heat in the T/R modules. This is the dominant thermal load on the payload and directly drives the radiator area computed in m3-1.

THE POWER-THERMAL COUPLING A 1,000-element Ka-band AESA at 3 W per T/R module = 3 kW DC input. At 35% PAE, the radiated RF power is 1.05 kW and the waste heat is 1.95 kW — plus HPA heat. The payload thermal subsystem must reject roughly 2–3 kW of heat from the T/R module array alone, requiring multiple square metres of dedicated radiator area as computed in m3-1.

5. Digital Beamforming

In a digital beamforming (DBF) array, the received signal at each element is digitised by an ADC, and beams are formed in the digital domain by applying complex weights and summing — equivalent to analogue phase-shifting and combining, but in software. The key advantage: once signals are digitised, any number of beams can be formed simultaneously by applying different weight vectors to the same element samples, limited only by available processing power.

Additional DBF advantages: adaptive beamforming (weights updated millisecond-by-millisecond in response to interference or demand), and precision calibration of element amplitude and phase in the digital domain, correcting manufacturing tolerances that would degrade analogue array performance.

The principal challenge is ADC data rate: a 1,000-element array at Ka-band with 500 MHz per element generates many Tbps of digital data that must be processed before beam formation. Managing this data volume — through subarray grouping, reduced sampling, and efficient interconnect — is the key DBF engineering problem. Telesat Lightspeed (MDA Aurora), and likely Starlink v2 and Kuiper, all incorporate some form of DBF.

DBF vs ANALOGUE: THE BEAM COUNT DIFFERENCE An analogue phased array forming N simultaneous beams requires N complete analogue receive chains — one per beam. A DBF array needs only one ADC per element; additional beams are formed in software with no additional RF hardware. For a 100-beam LEO broadband satellite, this difference is decisive in terms of mass and cost.

6. User-Link and Feeder-Link Antennas

User-link antennas face nadir and generate the spot beams serving user terminals. For Ka-band LEO broadband with 0.3°–0.7° spot beams, the user-link aperture is typically 1–3 metres effective diameter. Active phased arrays with DBF are the current architecture of choice: many simultaneous beams, electronic steering as the satellite moves, and adaptive capacity allocation across beams.

Feeder-link antennas connect to gateways (at higher frequencies — V-band: 47.2–51.4 GHz uplink, 37.5–42.5 GHz downlink — to access additional bandwidth) or to other satellites via ISL. Optical ISL terminals use laser beams with very narrow divergence, providing 10–100 Gbps per link with no RF spectrum coordination requirement. This is why optical ISLs are strongly preferred over RF ISLs for high-throughput constellation mesh networking.

7. Beam Scan Coverage Over the Orbital Arc

At 550 km with ε_min = 10°, the maximum Earth-central angle is approximately 59°, meaning the user-link antenna must scan to within 59° of nadir. The element pattern scan loss at 59° is cos(59°) ≈ 0.52, a loss of approximately −2.8 dB. At ε_min = 35° (typical Ka-band operational minimum), the scan angle is ~33° and the scan loss is cos(33°) ≈ 0.84, or about −0.75 dB — manageable.

Increasing ε_min actually improves link-budget uniformity: users near the edge of coverage see better elevation geometry, lower atmospheric path, and the scan loss is smaller. The cost is a smaller coverage footprint per satellite, requiring more satellites for continuous coverage — the fundamental tradeoff quantified in m2-4.

SCAN LOSS IN THE LINK BUDGET The ~2.8 dB scan loss at the edge of coverage (ε = 10°) must appear as a satellite antenna gain reduction in the link budget for edge users. This is one reason LEO broadband systems typically specify ε_min = 25°–40° in practice: the scan loss at lower elevations degrades link margin enough that the extra coverage area is not worth the cost of additional satellite power to compensate.

8. Pointing Within the Beam: Electronic Correction

As established in m3-2, the satellite has a residual mechanical pointing error of 0.03°–0.10° three-sigma. An active phased array with DBF corrects this electronically: a small differential phase gradient across the array shifts the beam direction by a fraction of the beamwidth, correcting the pointing offset in the RF domain without any mechanical adjustment. This electronic correction loop — fed by the attitude error from the ADCS star tracker — is standard on modern LEO broadband active array payloads and is one reason why the 0.1° three-sigma mechanical pointing requirement is achievable despite structural flex, thermal deformation, and control loop residuals.

For next-generation V-band and E-band systems with beamwidths of 0.1°–0.2°, this electronic correction becomes the primary pointing mechanism — the mechanical pointing system provides only coarse stabilisation, and the antenna electronics provide the fine correction needed to hold a 0.05° beam on target.

Exercise 1 — Array Gain and Beamwidth

Compute gain and HPBW for phased arrays of different element counts at Ka-band.

import math

def array_gain(n, g_elem_dbi, efficiency=0.65):
    """Array gain in dBi, with efficiency factor."""
    return g_elem_dbi + 10*math.log10(n) + 10*math.log10(efficiency)

def hpbw(aperture_m, freq_ghz):
    """HPBW in degrees for a uniformly illuminated aperture."""
    lam = 0.3 / freq_ghz
    return math.degrees(0.886 * lam / aperture_m)

freq = 28.0; g_elem = 6.0   # Ka-band, patch element
lam  = 0.3 / freq            # wavelength in metres

print(f"Ka-band {freq} GHz  λ={lam*1000:.1f} mm  element gain={g_elem} dBi\n")
print(f"{'N':>6}{'Aperture':>12}{'Gain (dBi)':>12}{'HPBW (°)':>10}")
print("-" * 42)
for n in [64, 256, 512, 1024, 2048, 4096]:
    side = math.sqrt(n) * lam / 2   # square array, λ/2 spacing
    g    = array_gain(n, g_elem)
    bw   = hpbw(side, freq)
    print(f"{n:>6}{side:>11.2f}m{g:>11.1f} dBi{bw:>9.2f}°")

Exercise 2 — Beam Steering and Scan Loss

Compute element phase shifts for beam steering and scan loss as a function of steering angle.

import math

def steer_phases(n, d_over_lam, theta_deg):
    """Phase shifts (rad) to steer n-element array to theta_deg."""
    t = math.radians(theta_deg)
    kd = 2 * math.pi * d_over_lam
    return [-i * kd * math.sin(t) for i in range(n)]

def scan_loss_db(theta_deg):
    """Patch element pattern scan loss in dB."""
    return 10 * math.log10(max(math.cos(math.radians(theta_deg)), 1e-9))

print("Scan loss vs steering angle (patch element pattern)\n")
print(f"{'Angle':>8}{'Scan loss':>12}{'Notes':>28}")
print("-" * 50)
for t in [0, 10, 20, 30, 33, 40, 50, 59, 65, 70]:
    loss = scan_loss_db(t)
    note = ""
    if t == 33: note = "← ε_min=35° edge"
    if t == 59: note = "← ε_min=10° edge"
    print(f"{t:>6}°{loss:>11.2f} dB{note:>28}")

print("\n8-element array steered to 30° (λ/2 spacing):")
phases = steer_phases(8, 0.5, 30.0)
for i, p in enumerate(phases):
    print(f"  elem {i}: {p:+.3f} rad ({math.degrees(p):+.1f}°)")

Exercise 3 — Multibeam Coverage Tiling

Compute how many simultaneous beams are needed to tile the coverage footprint for different minimum elevation angles and beam sizes.

import math

RE = 6378.137

def cap_radius_km(alt_km, eps_deg):
    r = RE + alt_km; e = math.radians(eps_deg)
    rho = math.acos(max(-1.,min(1., RE*math.cos(e)/r))) - e
    return RE * rho

def beam_radius_km(alt_km, hpbw_deg):
    return alt_km * math.tan(math.radians(hpbw_deg / 2))

def n_beams(cap_r, beam_r, overlap=0.7):
    return math.ceil(math.pi * cap_r**2 / (math.pi * (beam_r*overlap)**2))

alt = 550
print(f"Beam tiling at {alt} km altitude\n")
print(f"{'ε_min':>7}{'Cap (km)':>10}{'HPBW':>8}{'Beam r (km)':>13}{'Beams':>8}")
print("-" * 48)
for eps in [10, 25, 35]:
    cap_r = cap_radius_km(alt, eps)
    for hpbw in [0.3, 0.5, 1.0]:
        br = beam_radius_km(alt, hpbw)
        nb = n_beams(cap_r, br)
        print(f"{eps:>6}°{cap_r:>9.0f} km{hpbw:>7.1f}°{br:>12.0f} km{nb:>8}")
    print()
