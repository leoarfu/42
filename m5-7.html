MODULE 5
AIR INTERFACE & ACCESS
CAPSTONE

Module 4 ended with a promise: its engine produced C/(N₀+I₀) in dBHz and deferred the conversion to bits. This lesson delivers the conversion, assembled from six stages: spectrum becomes symbols through roll-off (m5-1), symbols become protected bits through the MODCOD ladder (m5-2), the carrier is shared among users (m5-3), the framing takes its cut (m5-4), and the same machinery re-evaluates for the NR branch (m5-5) and the direct-to-device corner (m5-6). The module's output is one function — net throughput per beam as a function of achieved C/(N₀+I₀), terminal class, and spectrum — and this lesson evaluates it end to end, measures where its leverage lives, and integrates it against rain statistics to produce the module's capstone artefact: the availability-throughput curve.

Two results from that evaluation are worth flagging before the arithmetic, because both correct intuitions this program itself taught earlier. First, m4-7's sensitivity rule — a dB gained anywhere is worth roughly one MODCOD rung — is true only below ladder saturation; a link that already selects the top rung gets nothing for another dB of C/N₀, and its only remaining capacity levers are the ones that raise the symbol rate or cut overhead. Second, for such a link, ACM converts rain from a capacity problem into a percentile problem: the expected annual throughput is within a rounding error of clear-sky, and what rain actually costs is the peak rate during the worst hours. Both results fall directly out of the assembled chain, and both change how a system engineer should argue about margin.

5. Sensitivity: where the leverage lives

Each lever moved independently against the clear-sky forward baseline (874.6 Mbps):

Lever Change Net effect
C/N₀+1 dB+0.0% (ladder saturated)
Rain fade−7.5 dB−16.8% (one rung)
Roll-off0.20 → 0.05+14.3%
Pilotson → off+2.2%
Scheduler0.97 → 0.98+1.0%
Selection margin1.5 → 0.5 dB+0.0% (no rung crossed here)

The table's structure is the lesson. This link sits at the top of the DVB-S2 ladder, and above saturation the currency changes: decibels of C/N₀ — the entire subject of Module 4 — buy nothing, while the symbol-rate and overhead levers (roll-off above all) buy their full face value. Below saturation the picture changes twice over: a dB near a rung boundary buys 5–20% — and roll-off's 14.3% of extra symbols can even go negative, because tightening α from 0.20 to 0.05 dilutes Es/N₀ by ≈ 0.6 dB (more symbols, same power), and if that crosses a rung boundary downward the rung loss beats the symbol gain. The exercise-1 harness exhibits exactly this at the mid-ladder operating point. The reconciliation with m4-7's sensitivity doctrine: a dB is worth a rung only while there are rungs left to climb — and this is precisely why DVB-S2X extended the ladder upward (m5-4 §6): 64-APSK and beyond exist to give hot gateway links somewhere to spend their excess decibels. A system engineer should ask, of any proposed link improvement, one question first: where on the ladder does this link sit?

6. The availability-throughput curve

Integrating the chain against the m4-4 rain statistics (zone F, corrected model) produces the module's capstone: net throughput as a function of time percentile. For the worked forward link, the curve is nearly flat: combined degradations (fade plus noise-rain coupling, recombined with the interference floor — the full m4-7 chain, which the curve model carries) up to the link's 6.3 dB saturation headroom are absorbed silently, and the zone-F statistics place that crossing at the 99.90th percentile. The curve is flat to three nines, one rung down (−16.8%) in the worst 0.1%, and the expected annual throughput computes to ≥ 99.9% of clear-sky net.

State the result carefully, because both halves matter. For a link with headroom, ACM makes rain an availability-of-peak problem, not a capacity problem — annual capacity planning can use the clear-sky figure, and the margin debate is really about the worst percentile's user experience. But the result is conditional on where the link sits: a cost-optimised link sized mid-ladder (the m4-4 3–4 dB-margin design) crosses rungs on every moderate fade, its curve slopes visibly, and its expected annual throughput sits measurably below clear-sky. The availability-throughput curve — not the single clear-sky number, not the single 99.9% number — is the honest characterisation of a beam, and the spine engine now draws it.

7. From beam to constellation, and what the engine exports

Beam throughput multiplies toward system capacity through factors this module deliberately does not own: beams per satellite and frequency reuse (Module 3's payload), beam-hopping duty cycle where illumination is time-shared (m5-4 §6's superframe as the dwell quantum), satellites in view and handover continuity (Module 2's geometry), and gateway/feeder capacity (Module 8's ground segment). The naive product — beam rate × beam count × reuse factor — is an upper bound that every one of those factors erodes; producing the eroded, honest figure is precisely the work of the modules ahead.

What Module 5 exports downstream, as declared interfaces: net throughput per beam per condition (the curve, both directions, all three terminal classes) to the capacity chain; and the latency components — m5-3's grant delay (~25–65 ms LEO DAMA) and m5-5's RTTs (~25.8 ms LEO transparent, ~541 ms GEO) plus the per-UE HARQ ceiling — to Module 7's end-to-end latency budget.

8. What the engine still ignores

Stated plainly, so no one mistakes the model for the world: interference enters only as the m4-6 C/I input, not as a coupled function of the reuse pattern and scheduler decisions; traffic is a packet-size constant, not a demand process — no diurnal load, no statistical multiplexing gain, no congestion; the scheduler is an efficiency scalar, not a queueing system; mobility appears only as HARQ RTT, with handover interruption and beam-transition losses unmodelled; rain is spatially uncorrelated across beams, which real weather is not; and the NR branch rides a ±tens-of-percent link abstraction (m5-5 §5). Each omission is a deliberate scope boundary with a forwarding address in Modules 6–9.

10. Module closing

Module 5's question was the conversion of decibel-hertz into bits per second, and the answer is now an executable object: a throughput engine whose every factor was derived in a lesson, verified numerically, and reconciled against double counting — from m5-1's symbol rate to m5-6's frozen handset, across two air-interface families and three terminal classes. MODCOD = [("QPSK 1/4",0.490243,-2.35),("QPSK 1/2",0.988858,1.00),
          ("QPSK 3/4",1.487473,4.03),("8PSK 3/4",2.228124,7.91),
          ("16APSK 2/3",2.637201,8.97),("16APSK 3/4",2.966728,10.21),
          ("32APSK 3/4",3.703295,12.73),("32APSK 9/10",4.453027,16.05)]
PILOT = {"QPSK":0.9762,"8PSK":0.9773,"16APSK":0.9763,"32APSK":0.9784}

def net(cn0, bw=250e6, alpha=0.20, sched=0.97, gse=0.9934,
        margin=1.5, pilots=True):
    rs = bw/(1+alpha)
    es = cn0 - db(rs)
    best = None
    for n,e,t in MODCOD:
        if es >= t+margin and (best is None or e > best[1]): best = (n,e)
    if not best: return 0.0
    p = PILOT[[k for k in PILOT if best[0].startswith(k)][0]] if pilots else 1.0
    return best[1]*rs*sched*p*gse

for label, cn0 in [("LADDER-TOP link", 107.1), ("MID-LADDER link", 94.0)]:
    base = net(cn0)
    print(f"\n{label}  (C/N0 {cn0} dBHz, base {base/1e6:.1f} Mbps)")
    levers = [("+1 dB C/N0",        net(cn0+1)),

              ("pilots off",        net(cn0, pilots=False)),

Exercise 2 — The availability-throughput curve

Integrate the chain against zone-F rain statistics; produce the module's capstone number.

import math
db = lambda x: 10*math.log10(x)
# paste net() and tables from exercise 1 before running

def rain_f(avail, R001=28, f=20, elev=30):        # m4-4 corrected model
    p = max(100-avail, 0.01); e = math.radians(elev)
    Ls = 3.9/math.sin(e); L0 = 35*math.exp(-0.015*R001)
    k, a = 0.0751, 1.099
    A0 = k*R001**a * Ls * (1/(1+Ls*math.cos(e)/L0))
    if p == 0.01: return A0
    return A0*(p/0.01)**(-(0.655+0.033*math.log(p)-0.045*math.log(A0)))

def combined_degradation(fade_db, t_clear=142.0, ci0=109.0, cn0_th_clear=111.5):
    """Full m4-7 chain: fade + noise-rain coupling on the thermal C/N0,
    then recombination with the interference floor. Rain fade alone
    UNDERSTATES the hit (and, in clear-interference-limited links,
    the recombination partially compresses it back)."""
    t = t_clear + 273*(1 - 10**(-fade_db/10))
    cn0_th = cn0_th_clear - fade_db - db(t/t_clear)
    comb = -db(10**(-cn0_th/10) + 10**(-ci0/10))
    clear = -db(10**(-cn0_th_clear/10) + 10**(-ci0/10))
    return clear - comb          # dB of combined degradation

def annual(cn0_clear, **kw):
    pts = [50, 95, 99, 99.5, 99.9, 99.99, 99.999]
    total = pts[0]/100 * net(cn0_clear, **kw)          # clear below median
    for i in range(len(pts)-1):
        frac = (pts[i+1]-pts[i])/100
        deg = combined_degradation(rain_f((pts[i]+pts[i+1])/2))
        total += frac * net(cn0_clear - deg, **kw)
    return total

for label, cn0 in [("ladder-top", 107.1), ("mid-ladder", 94.0)]:
    clear = net(cn0); exp = annual(cn0)
    print(f"{label:<11} clear {clear/1e6:7.1f} Mbps | annual expectation "
          f"{exp/1e6:7.1f} Mbps = {exp/clear*100:5.1f}% of clear-sky")
print("\npercentile curve (ladder-top), full degradation:")
for a in [50, 95, 99, 99.5, 99.9, 99.99]:
    deg = combined_degradation(rain_f(a))
    print(f"  {a:>6}%: fade {rain_f(a):5.2f} dB -> deg {deg:5.2f} dB"
          f" -> {net(107.1-deg)/1e6:7.1f} Mbps")

Exercise 3 — Spine final assembly: THE THROUGHPUT ENGINE

All six stages, both branches, three terminal classes, one object. The module's public artefact.

import math
db = lambda x: 10*math.log10(x)

# ═══════════════════════════════════════════════════════
# M5 SPINE PROJECT — THROUGHPUT ENGINE (final assembly)
#   Stage 1  carrier      (m5-1): spectrum, roll-off -> Rs
#   Stage 2  MODCOD       (m5-2): Es/N0 -> eta, margins
#   Stage 3  access       (m5-3): sched eff / MF-TDMA bursts
#   Stage 4  framing      (m5-4): pilots + GSE (Table-13
#            reconciliation: BB/PLHEADER already in eta)
#   Stage 5  NR branch    (m5-5): RE ledger, beam vs UE-peak
#   Stage 6  D2D corner   (m5-6): frozen handset
#   This stage: assembly, availability curve, M7 exports.
# ═══════════════════════════════════════════════════════

MODCOD = [("QPSK 1/4",0.490243,-2.35),("QPSK 1/2",0.988858,1.00),
          ("QPSK 3/4",1.487473,4.03),("8PSK 3/4",2.228124,7.91),
          ("16APSK 2/3",2.637201,8.97),("16APSK 3/4",2.966728,10.21),
          ("32APSK 3/4",3.703295,12.73),("32APSK 9/10",4.453027,16.05)]
PILOT = {"QPSK":0.9762,"8PSK":0.9773,"16APSK":0.9763,"32APSK":0.9784}
NRB   = {(20,15):106,(50,30):133,(100,30):273,(400,120):264}

class ThroughputEngine:
    def __init__(self, name, margin_db=1.5):
        self.name, self.margin = name, margin_db

    # ── stage 2 ──
    def _select(self, esn0):
        best = None
        for n,e,t in MODCOD:
            if esn0 >= t+self.margin and (best is None or e > best[1]):
                best = (n,e,t)
        return best

    # ── DVB forward: stages 1-4 ──
    def dvb_forward(self, bw_hz, alpha, cn0, sched=0.97,
                    pilots=True, pkt=1500):
        rs = bw_hz/(1+alpha)
        s  = self._select(cn0 - db(rs))
        if not s: return dict(net=0.0, modcod=None)
        p  = PILOT[[k for k in PILOT if s[0].startswith(k)][0]] if pilots else 1.0
        g  = pkt/(pkt+10)
        return dict(modcod=s[0], info=s[1]*rs,
                    net=s[1]*rs*sched*p*g)

    # ── DVB return: stages 1-4 with the m5-3 burst model ──
    def dvb_return(self, bw_hz, n_car, alpha, cn0_per_carrier,
                   slot_ms=2.0, oh_sym=300, contention=0.05):
        rc = (bw_hz/n_car)/(1+alpha)
        s  = self._select(cn0_per_carrier - db(rc))
        if not s: return dict(net=0.0, modcod=None)
        syms  = rc*slot_ms/1e3
        burst = (syms-oh_sym)/syms
        per   = s[1]*rc*burst*(1-contention)
        return dict(modcod=s[0], per_carrier=per, beam=per*n_car)

    # ── NR branch: stage 5 ──
    def nr_beam(self, bw_mhz, scs_khz, cn0, ctrl=0.14):
        nrb = NRB[(bw_mhz, scs_khz)]
        occ = nrb*12*scs_khz*1e3
        snr = cn0 - db(occ)
        mu  = int(math.log2(scs_khz/15)); slot = 1.0/2**mu
        eta = min(7.406, 0.75*math.log2(1+10**(snr/10)))
        beam = nrb*12*14/(slot*1e-3)*(1-ctrl)*eta
        return dict(beam=beam, slot_ms=slot,
                    ue_peak=lambda n_harq, rtt: beam*min(1, n_harq*slot/rtt))

    # ── D2D corner: stage 6 ──
    def d2d_beam(self, array_m2, eirp_dbw, spectrum_hz=5e6,
                 f_ghz=2.0, slant_km=1000, losses=4.0):
        fspl = 92.45 + 20*math.log10(slant_km) + 20*math.log10(f_ghz)
        lam  = 0.3/f_ghz
        gt_sat = db(4*math.pi*array_m2/lam**2*0.7) - db(500)
        dl = eirp_dbw - fspl - losses - 30.0 + 228.6     # handset G/T -30
        ul = -10.0    - fspl - losses + gt_sat + 228.6   # handset EIRP -10
        cn0 = min(dl, ul)
        for bw in [5e6, 1.4e6, 720e3, 360e3, 180e3]:
            if cn0 - db(bw) >= 4.0:
                n = int(spectrum_hz//bw)
                snr = cn0 - db(n) - db(bw)               # power split
                eta = min(7.406, 0.75*math.log2(1+10**(snr/10)))
                return dict(beam=n*bw*eta, dl_cn0=dl, ul_cn0=ul)
        return dict(beam=0.0, dl_cn0=dl, ul_cn0=ul)

    # ── the capstone: availability-throughput curve ──
    @staticmethod
    def _combined_degradation(fade_db, t_clear=142.0, ci0=109.0,
                              cn0_th_clear=111.5):
        # Full m4-7 chain: fade + noise-rain coupling, recombined
        # with the interference floor. Fade alone understates the hit.
        t = t_clear + 273*(1 - 10**(-fade_db/10))
        cn0_th = cn0_th_clear - fade_db - db(t/t_clear)
        comb  = -db(10**(-cn0_th/10) + 10**(-ci0/10))
        clear = -db(10**(-cn0_th_clear/10) + 10**(-ci0/10))
        return clear - comb

    def availability_curve(self, cn0_clear, bw_hz, alpha,
                           rain_fn, points=(50,95,99,99.5,99.9,99.99)):
        rows = []
        for a in points:
            deg = self._combined_degradation(rain_fn(a))
            rows.append((a, deg,
                         self.dvb_forward(bw_hz, alpha, cn0_clear-deg)))
        return rows

# ── zone-F rain (m4-4 corrected) ────────────────────────
def rain_f(avail, R001=28):
    p = max(100-avail, 0.01); e = math.radians(30)
    Ls = 3.9/math.sin(e); L0 = 35*math.exp(-0.015*R001)
    A0 = 0.0751*R001**1.099*Ls/(1+Ls*math.cos(e)/L0)
    if p == 0.01: return A0
    return A0*(p/0.01)**(-(0.655+0.033*math.log(p)-0.045*math.log(A0)))
