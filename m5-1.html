Module 4 ended with a number: the achieved C/(N₀+I₀) of a link, in dBHz. That number says nothing about bits. A link budget describes how much signal power arrives relative to the noise floor; it does not say how many bits per second that power can carry, because the answer depends entirely on how the bits are encoded onto the radio wave. Module 5 builds that conversion — from dBHz to bits per second per beam — and this lesson builds its first stage: the mapping from a bit stream to a physical waveform occupying a definite slice of spectrum.

The chain this module assembles is: waveform (this lesson) → modulation and coding selection (m5-2) → sharing the channel among users (m5-3) → framing and the real air interfaces that package all of it (m5-4 through m5-6) → net throughput per beam (m5-7). The Module 5 spine project — a Python throughput engine that consumes the Module 4 link budget output — begins here with its first stage: a carrier model that converts symbol rate, roll-off, and modulation order into occupied bandwidth and raw bit rate.

Two ideas dominate this lesson. First, bandwidth is spent on symbols, not bits — the number of bits each symbol carries is a design choice with a steep price in required signal power. Second, the satellite channel is nonlinear in a way terrestrial channels are not, and this single fact explains why satellite air interfaces use constellations (APSK) that almost no terrestrial system uses.

1. From bits to a passband waveform

A digital transmitter must convert a stream of bits into a continuous radio-frequency signal. The universal representation is the quadrature (I/Q) form:

s(t) = I(t) · cos(2πf_c t) − Q(t) · sin(2πf_c t)

where f_c is the carrier frequency and I(t), Q(t) are two independent baseband signals — the in-phase and quadrature components. Because cosine and sine are orthogonal, a receiver can recover I and Q independently. Every modulation scheme in this module is just a rule for choosing the pair (I, Q) as a function of the input bits.

The transmitter groups the bit stream into blocks of k bits, and each block selects one of M = 2^k allowed (I, Q) pairs. Each selected pair is held (after pulse shaping — section 3) for one symbol duration T_s. The pair is a symbol; the set of M allowed pairs, plotted on the I/Q plane, is the constellation.

The two rates that matter:

Symbol rate: R_s = 1 / T_s [symbols per second, "baud"]
Raw bit rate: R_b = k · R_s = log₂(M) · R_s [bits per second]

The channel — the amplifier, the antenna, the atmosphere, the spectrum allocation — sees only symbols. Bandwidth is consumed by R_s. The bit rate is R_s multiplied by however many bits the designer dares to pack into each symbol. That "dares" is the subject of sections 5 and 6.

2. Constellations: PSK, APSK, QAM

Three constellation families cover essentially all of satellite communications:

Phase-shift keying (PSK) places all M points on a single circle, encoding information in phase only. All symbols have identical amplitude.

  • BPSK (M=2, 1 bit/symbol): two points, 180° apart. The most robust scheme in existence; used for TT&C, GPS signals, and the lowest rungs of adaptive ladders.
  • QPSK (M=4, 2 bits/symbol): four points, 90° apart. The workhorse of satellite communications for fifty years.
  • 8-PSK (M=8, 3 bits/symbol): eight points, 45° apart. The points crowd together; noise tolerance drops noticeably.

Amplitude and phase-shift keying (APSK) places points on concentric rings. 16-APSK uses a 4+12 arrangement — 4 inner points, 12 outer. 32-APSK uses 4+12+16 on three rings. The ratio between ring radii is a design parameter, and in DVB-S2/S2X it is specified per code rate so that the constellation shape is matched to the decoder's operating point.

Quadrature amplitude modulation (QAM) places points on a square grid: 16-QAM, 64-QAM, 256-QAM. QAM maximises the minimum distance between points for a given average power, which makes it the standard for terrestrial systems — Wi-Fi, cable, cellular, microwave links all use QAM. Satellite systems largely do not, for the reason developed in section 5.

One convention worth fixing now: bit-to-symbol mapping uses Gray coding, in which adjacent constellation points differ in exactly one bit. When noise pushes a received symbol into a neighbouring decision region — by far the most likely error — only one bit is corrupted. Gray coding costs nothing and roughly halves the bit error rate relative to naive mapping; every practical system uses it.

3. Bandwidth: Nyquist, roll-off, and occupied spectrum

An idealised symbol — a rectangular pulse held for T_s — occupies infinite bandwidth. Real systems shape each symbol with a filter so that the spectrum is confined and, critically, so that symbols do not smear into each other at the sampling instants (zero inter-symbol interference). The standard choice is the root-raised-cosine (RRC) filter, applied half at the transmitter and half at the receiver, characterised by a single parameter: the roll-off factor α.

The occupied bandwidth of an RRC-shaped carrier is:

B_occ = R_s · (1 + α)

The Nyquist limit is α = 0: a symbol rate of R_s fits, in theory, in exactly R_s Hz. Real filters need excess bandwidth. DVB-S2 specified α ∈ {0.20, 0.25, 0.35}; DVB-S2X extended it down to {0.05, 0.10, 0.15}, because in a bandwidth-limited system, every percent of roll-off is a percent of capacity.

Worked numbers for a 250 MHz Ka-band channel allocation:

Roll-off α Symbol rate R_s in 250 MHz Gain vs α = 0.20
0.20208.3 Msym/s—
0.05238.1 Msym/s+14.3%

Tightening the roll-off from 0.20 to 0.05 puts 14.3% more symbols — and therefore 14.3% more bits, at any modulation — through the same allocation. This is free capacity in the sense that it costs no power and no spectrum; it costs filter complexity, sensitivity to timing error, and a higher peak-to-average power ratio, which feeds directly into the amplifier problem of section 5.

4. Raw spectral efficiency

Combining sections 2 and 3, the raw (pre-coding) spectral efficiency of a carrier is:

η_raw = R_b / B_occ = log₂(M) / (1 + α) [bit/s/Hz]

For the same 250 MHz channel at α = 0.20 (R_s = 208.3 Msym/s):

Scheme Raw bit rate η_raw
QPSK416.7 Mbps1.67 bit/s/Hz
16-APSK833.3 Mbps3.33 bit/s/Hz
32-APSK1,041.7 Mbps4.17 bit/s/Hz

These figures are deliberately labelled raw. They assume every transmitted bit is a payload bit. In reality a fraction of the bits are forward error correction parity (m5-2) and a further fraction are framing, pilots, and headers (m5-4). Net spectral efficiency is always lower — often by a factor of 1.5–3. Quoting raw rates as if they were delivered capacity is one of the oldest tricks in satellite marketing; the spine project keeps raw and net strictly separate for exactly this reason.

5. The nonlinear channel: why satellite uses APSK, not QAM

On the I/Q plane, QAM is the better constellation: for a fixed average power, the square grid maximises the minimum distance between points, which minimises error probability in Gaussian noise. Terrestrial systems therefore use QAM everywhere. Satellite systems use APSK instead, and the reason is the high-power amplifier (HPA) on the satellite.

A satellite HPA — travelling-wave tube or solid-state — is most power-efficient when driven at or near saturation, its maximum output. Near saturation the amplifier is strongly nonlinear: output amplitude compresses (AM/AM distortion) and output phase shifts with input amplitude (AM/PM distortion). A signal whose envelope fluctuates gets distorted in proportion to those fluctuations.

The envelope fluctuation of a modulated carrier is measured by its peak-to-average power ratio (PAPR). Constellations with many amplitude levels — QAM above all — have high PAPR. The operator's escape is output back-off (OBO): run the amplifier below saturation, in its linear region. But every dB of back-off is a dB of EIRP surrendered, and Module 4 established what a dB of EIRP is worth. On a power-limited spacecraft, backing a 100 W amplifier off by 6 dB means radiating 25 W.

Typical operating points (ranges; exact values depend on amplifier linearisation and the DVB-S2 implementation guidelines' methodology):

Constellation Amplitude rings Typical OBO required
QPSK1≈ 0–1 dB (near saturation)
16-APSK2≈ 2–4 dB
32-APSK3≈ 3–5 dB
16-QAM3 (grid)≈ 5–7 dB

APSK is the compromise this table explains. By restricting points to a small number of rings — two for 16-APSK, three for 32-APSK — APSK keeps the envelope variation low enough that the amplifier can run only a few dB from saturation. The ring-radius ratio is then optimised (per code rate, in DVB-S2) to recover most of the noise-distance QAM would have offered. APSK is a constellation designed for the amplifier first and the noise second — a priority ordering that only makes sense when the amplifier rides on a spacecraft where every watt is precious.

Two refinements complete the picture. Pre-distortion — applying the inverse of the amplifier's nonlinearity at the modulator — claws back 1–2 dB of the back-off and is standard in modern gateways. And the trend toward higher-order APSK (64-APSK, 256-APSK in DVB-S2X) is only usable on links with both high C/N and well-linearised amplifiers, which in practice means gateway feeder links far more often than user downlinks.

6. The power cost of density: uncoded error performance

Packing more bits per symbol crowds the constellation points closer together at fixed average power, so the same noise causes more errors. The currency for comparing schemes is the Eb/N₀ required for a target bit error rate (BER) — energy per bit over noise density, which Module 4 related to C/N₀ via Eb/N₀ = C/N₀ − 10·log₁₀(R_b).

For coherent detection with Gray coding in additive white Gaussian noise, QPSK has the classic closed form:

P_b = Q(√(2 · Eb/N₀))

where Q is the Gaussian tail function. Numerically solved thresholds at BER = 10⁻⁵, uncoded:

Scheme Required Eb/N₀ (uncoded, BER 10⁻⁵)
BPSK / QPSK9.6 dB
8-PSK≈ 13.0 dB
16-QAM≈ 13.4 dB

Note the QPSK result carefully: BPSK and QPSK require the same Eb/N₀. QPSK carries twice the bits per symbol, but each bit rides on an independent quadrature, so per-bit performance is identical. QPSK is strictly better than BPSK in spectral efficiency at zero power cost — which is why BPSK survives only where extreme robustness or implementation simplicity dominates.

The ladder above is the uncoded ladder, and no modern system operates uncoded. Its role in this module is to establish the shape of the trade: each step up in constellation density buys bits per symbol and costs dB of required Eb/N₀, and the exchange rate worsens as the constellation grows. Lesson m5-2 shows how forward error correction shifts the entire ladder 5–9 dB to the left — and turns it into the MODCOD table that the spine project's selection logic runs on.

7. The Shannon ceiling

There is a hard upper bound on what any waveform, any code, and any amount of engineering can extract from a channel. The Shannon–Hartley capacity of a bandwidth B with signal-to-noise ratio SNR is:

C = B · log₂(1 + SNR) [bit/s]

Equivalently, transmitting at spectral efficiency η bit/s/Hz requires at minimum:

SNR_min = 2^η − 1, (Eb/N₀)_min = (2^η − 1) / η
Target η (bit/s/Hz) Minimum SNR Minimum Eb/N₀
10.0 dB0.0 dB
24.8 dB1.8 dB
38.5 dB3.7 dB
411.8 dB5.7 dB
514.9 dB7.9 dB
618.0 dB10.2 dB

As η → 0 the required Eb/N₀ approaches the ultimate floor of −1.59 dB: below this, no communication is possible at any rate by any means.

The table quantifies the opportunity that motivates the rest of this module. Uncoded QPSK delivers ~2 bit/s/Hz at 9.6 dB Eb/N₀; Shannon says 2 bit/s/Hz is achievable at 1.8 dB. The gap — roughly 7.8 dB — is the territory that forward error correction conquers. Modern LDPC codes (m5-2) close to within about 1 dB of the bound, which is why the MODCOD thresholds quoted in m4-7 (85 dBHz for QPSK 1/2, 105 dBHz for 16-APSK 3/4 at their respective rates) sit where they do.

Shannon capacity also gives the spine project its sanity check: for any C/N the Module 4 engine reports, the engine can compute the Shannon ceiling for the channel bandwidth, and every throughput figure the M5 engine ever produces must sit below it. A model output above the Shannon line is not optimistic; it is wrong.

9. What this lesson delivers to the design chain

Module 4 produced C/(N₀+I₀). This lesson converts a spectrum allocation into a symbol rate (via roll-off) and a symbol rate into a raw bit rate (via constellation choice), and establishes the two constraints every later lesson operates inside: the power cost of constellation density, and the Shannon ceiling neither cleverness nor marketing can exceed. It also fixed why the satellite constellation alphabet is APSK — a decision made by the amplifier, not the mathematician.

The spine project now exists as Stage 1: a carrier model that maps (bandwidth, roll-off, modulation) → (symbol rate, raw bit rate, raw spectral efficiency) with a Shannon ceiling check. Lesson m5-2 attaches the missing organ: the forward error correction that turns raw bits into protected bits, and the MODCOD table that tells the engine which combination of modulation and code rate a given C/N₀ can actually support.

Exercise 1 — The uncoded Eb/N₀ ladder

Compute the required Eb/N₀ at a target BER for the PSK/QAM family from the Q-function, and observe the price of constellation density.

import math

def Q(x):
    """Gaussian tail probability."""
    return 0.5 * math.erfc(x / math.sqrt(2))

def ber_qpsk(ebn0_db):
    g = 10 ** (ebn0_db / 10)
    return Q(math.sqrt(2 * g))

def ber_8psk(ebn0_db):
    g = 10 ** (ebn0_db / 10)
    return (2/3) * Q(math.sqrt(6 * g) * math.sin(math.pi / 8))

def ber_16qam(ebn0_db):
    g = 10 ** (ebn0_db / 10); M, k = 16, 4
    return (4/k) * (1 - 1/math.sqrt(M)) * Q(math.sqrt(3 * k * g / (M - 1)))

def threshold(ber_fn, target=1e-5):
    """Bisect for the Eb/N0 (dB) meeting the target BER."""
    lo, hi = 0.0, 30.0
    for _ in range(100):
        mid = (lo + hi) / 2
        if ber_fn(mid) > target: lo = mid
        else: hi = mid
    return mid

print("Uncoded Eb/N0 thresholds at BER 1e-5 (AWGN, Gray-coded):")
for name, fn in [("BPSK/QPSK", ber_qpsk), ("8-PSK", ber_8psk), ("16-QAM", ber_16qam)]:
    print(f"  {name:<10} {threshold(fn):>6.2f} dB")

Exercise 3 — Spine Stage 1: The carrier model (seed)

Begin the Module 5 spine project — a Python throughput engine that will be extended in m5-2 through m5-7 and consumes the Module 4 link budget engine's output.

import math

# ═══════════════════════════════════════════════════════
# M5 SPINE PROJECT — THROUGHPUT ENGINE
# Stage 1: carrier model — bandwidth, roll-off, modulation
#          → symbol rate, raw bit rate, Shannon ceiling
# Each subsequent lesson adds one stage:
#   m5-2  MODCOD table + selection from achieved C/N0
#   m5-3  multiple-access overhead & scheduling efficiency
#   m5-4  framing overhead (DVB-S2X / proprietary)
#   m5-5  OFDM/NTN numerology branch
#   m5-6  D2D narrowband case
#   m5-7  full assembly: C/(N0+I0) → net beam capacity
# ═══════════════════════════════════════════════════════

MODULATIONS = {
    "BPSK":     1, "QPSK":     2, "8PSK":     3,
    "16APSK":   4, "32APSK":   5, "64APSK":   6, "256APSK": 8,
}

class Carrier:
    """
    Stage-1 carrier model. Raw quantities only — no FEC,
    no framing, no access overhead yet. Those stages bolt on
    in later lessons and only ever reduce the numbers here.
    """
    def __init__(self, name, bw_hz, alpha, modulation):
        if modulation not in MODULATIONS:
            raise ValueError(f"Unknown modulation: {modulation}")
        self.name       = name
        self.bw_hz      = bw_hz
        self.alpha      = alpha
        self.modulation = modulation
        self.k          = MODULATIONS[modulation]

    @property
    def symbol_rate(self):
        return self.bw_hz / (1 + self.alpha)

    @property
    def raw_bit_rate(self):
        return self.k * self.symbol_rate

    @property
    def raw_spectral_efficiency(self):
        return self.raw_bit_rate / self.bw_hz     # = k / (1+alpha)

    def shannon_capacity(self, cn0_dbhz):
        """
        Shannon ceiling for this carrier's bandwidth at the
        C/N0 the Module 4 engine reports. Hard upper bound —
        every later stage's output must sit below this.
        """
        cn_db  = cn0_dbhz - 10 * math.log10(self.bw_hz)
        snr    = 10 ** (cn_db / 10)
        return self.bw_hz * math.log2(1 + snr)

    def report(self, cn0_dbhz=None):
        print(f"\n{'─'*56}")
        print(f"  CARRIER — {self.name}")
        print(f"{'─'*56}")
        print(f"  Bandwidth            {self.bw_hz/1e6:>10.1f} MHz")
        print(f"  Roll-off α           {self.alpha:>10.2f}")
        print(f"  Modulation           {self.modulation:>10} ({self.k} bit/sym)")
        print(f"  Symbol rate          {self.symbol_rate/1e6:>10.1f} Msym/s")
        print(f"  RAW bit rate         {self.raw_bit_rate/1e6:>10.1f} Mbps")
        print(f"  RAW spectral eff.    {self.raw_spectral_efficiency:>10.2f} bit/s/Hz")
        if cn0_dbhz is not None:
            cap = self.shannon_capacity(cn0_dbhz)
            print(f"  Shannon ceiling      {cap/1e6:>10.1f} Mbps  (C/N0 {cn0_dbhz} dBHz)")
            if self.raw_bit_rate > cap:
                print(f"  ⚠ RAW rate exceeds Shannon ceiling at this C/N0 —")
                print(f"    this MODCOD cannot be supported; selection logic")
                print(f"    arrives in m5-2.")
        print(f"{'─'*56}")

# ── Example: Ka-band forward-link carrier ───────────────
# C/N0 input taken from the Module 4 spine engine output.
c = Carrier("Ka forward, user beam", bw_hz=250e6, alpha=0.10, modulation="16APSK")
c.report(cn0_dbhz=96.0)

c2 = Carrier("Same carrier, QPSK", bw_hz=250e6, alpha=0.10, modulation="QPSK")
c2.report(cn0_dbhz=96.0)
